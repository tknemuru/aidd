# api-overview.md — aidd インターフェースの設計意図

## 本ドキュメントの位置付け

本ドキュメントは、aidd が提供する CLI コマンド群および AI バックエンド抽象の設計意図を記述する。コマンド引数の一覧や AI バックエンドの詳細仕様はコード（`bin/src/`）と `README.md` を正として参照する。ここでは「なぜそのインタフェース形状か」「どのような不変条件を守るか」を明文化する。

## CLI コマンド群の設計方針

| 方針 | 内容 |
| :--- | :--- |
| 副作用は明示的な引数で制御 | 位置引数のみを受け付け、暗黙の環境解釈を減らす |
| 標準出力は機械可読、標準エラーは人間可読 | 呼び出し元スクリプトがパースする値は `stdout`、進行状況は `stderr` |
| 失敗時は非ゼロ終了 | 呼び出し元でパイプライン中断判定を一律で扱えるようにする |
| Git / gh 依存は最小限の薄いラップ | CLI ツール側に正本を持たせ、aidd 側で二重メンテナンスを避ける |

## AI バックエンド抽象の設計意図

| 設計項目 | 意図 |
| :--- | :--- |
| 契約を 1 メソッド（`run`）に集約 | バックエンド追加の工数を最小化し、抽象境界の破壊を防ぐ |
| `allowedTools` は配列 | バックエンド側で CLI オプション形式へ正規化しやすくする |
| `timeoutSec` をパラメータ化 | 呼び出し側がフェーズごとに安全弁を設定できるようにする |
| 実装の選択は `resolveBackend` に集約 | 環境変数解釈を 1 箇所に閉じ、誤選択を防ぐ |
| 未対応値は即時例外 | フォールバックによる「動いているように見える状態」を作らない |

## 共通パターン

### リカバリラッパ

- 最大 3 試行（既定）で再試行し、各試行でタイムアウトを適用する。
- 失敗時は前回の標準エラーと元プロンプトを連結した修復プロンプトで再試行する。
- いずれの試行でも標準出力にエスカレーションマーカーが検出された時点で即時中断する。

### エスカレーション表現

- マーカー `ESCALATION_REQUIRED` に続き、`ブロッカー` / `理由` / `推奨アクション` / `再開条件` の 4 項目を標準出力へ書き出す。
- 受け手（呼び出し元スクリプト・人間）はマーカーを文字列として検出し、プロセス終了コード 2 で中断される前提で設計する。

### プロンプト変換の検証

- 中立プロンプト本文に明示記載された `--allowedTools "..."` のリストを抽出し、`tool-map` の登録名と照合する。
- 未登録ツール名は `UnknownToolError` を送出して配布プロセスを中断させ、黙示的な欠落配布を防ぐ。

## エラーハンドリング規約

| 事象 | 扱い |
| :--- | :--- |
| AI_BACKEND 未対応値 | 例外送出、復旧しない |
| 中立プロンプトに未登録ツール名 | 例外送出、配布中断 |
| Git / gh サブコマンド失敗 | 非ゼロ終了、標準エラーに原因を出す |
| リカバリラッパのタイムアウト | 負値の `exitCode` としてラップに返す |
| フェーズ再判定でマップより先行 | 自動スキップ、決定ログに記録 |

## 認証方式

- aidd 自身はトークンを扱わない。AI / Git / gh の各 CLI が保持する認証状態をそのまま流用する。
- Claude / Copilot いずれのバックエンドも対話的ログインを前提とせず、事前にセッションが確立されていることをユーザ責務とする。
- 認証不備を AI セッション内で検出した場合はエスカレーションマーカーで人間に引き継ぐ。
