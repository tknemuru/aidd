# domain-model.md — aidd ドメイン知識

## 本ドキュメントの位置付け

本ドキュメントは、aidd が扱うドメイン概念・業務ルール・運用要件を AI エージェントに伝達する。コードの挙動は実装で確認し、ここでは「なぜその挙動であるべきか」「どのような前提の下で動くか」を明文化する。

## ドメイン概念

| 概念 | 意味 |
| :--- | :--- |
| 中立プロンプト | Claude / Copilot いずれにも変換可能なスラッシュコマンド本文の正本 |
| 配布資産 | 対象リポジトリに同期される Claude 向け / Copilot 向けのプロンプトおよび指示書 |
| slug | RFC / サービス仕様の識別子。日付プレフィクス付き完全 slug と、ユーザ入力 slug の 2 形式がある |
| ライフサイクルフェーズ | RFC / RRFC / MERGE_RFC / IMP / RIMP / VFY / MERGE_IMPL / DONE の 8 段階 |
| エスカレーション | AI が自力で完遂できない事象を検知し、人間へ引き継ぐ動作 |
| リスクレベル | 自動進行可否と人間承認要否を決定する属性（high / medium / low） |

## 業務ルール・制約

### バックエンド切替

- AI バックエンドは環境変数 `AI_BACKEND` により選択する。既定は `claude`。
- `claude` / `copilot` 以外の値は未対応として即時例外を送出する。フォールバックや推測による代替選択は行わない。
- バックエンドの振る舞いは同じ契約（ヘッドレス 1 問 1 答）に揃える。差異は呼び出し側で意識されない。

### 配布規約

- 中立プロンプトマスタは `adapters/commands/` を正本とする。Claude 向けは無加工コピー、Copilot 向けは frontmatter 付与と検証を伴う変換で生成する。
- Copilot 向け変換で未登録ツール名を検出した場合、配布を中断する。ツール名の追加は `tool-map.ts` への登録を伴う設計変更として扱う。
- 配布資産の参照経路は対象リポジトリの `.claude/` および `.github/prompts/` に完結させ、aidd リポジトリへの絶対パス依存を残さない。

### エスカレーション条件

- AI が認証情報を要求された場合、物理操作を要求された場合、あるいはビジネス判断が必要となった場合は即時停止する。
- ダミー値の注入・ステップのスキップ・仮完了の報告は禁止する。
- エスカレーションマーカー（`ESCALATION_REQUIRED`）を標準出力に含めて終了コード 2 で終了する。

### リスクレベル別ゲート通過

- GATE-0 / GATE-I0 は全リスクレベルで FAIL 時にエスカレーションする。
- high / medium では GATE-5 / GATE-2 / GATE-I1 / GATE-I2 の FAIL もエスカレーションする。
- low では上記以外は自動修正ループに乗せてよい。
- PR マージは high リスクでのみ人間承認を必須とする。
- 検証結果は high で常に、medium で FAIL のみ人間確認を必須とする。

### フェーズ判定の不変条件

- 成果物の状態は「PR のマージ状態」「レビュー結果ファイルの PASS 有無」「ブランチ存在」の 3 軸で決まる。
- 判定順序は `DONE → MERGE_IMPL → VFY → RIMP → IMP → MERGE_RFC → RRFC → RFC` の逆順とする。上位条件を満たす状態は下位判定より優先される。
- 再判定結果がフェーズマップより進んでいる場合はスキップする。後退はしない。

## 運用要件

- 配布 CLI は Windows ネイティブと Linux のどちらでも同一のコマンド名で起動可能であることを要求する。
- インストールは単一コマンド（`npm install -g .`）で完結することを要求する。
- エスカレーション発生時は自動ループを中断し、人間がレビュー PR を確認するまで次フェーズへ進まない。
- 自動開発オーケストレーション中の意思決定は `adev-decisions.md` に追記し、事後監査に耐える痕跡を残す。

## ドメイン固有の前提知識

- AI モデルは Claude 系を前提とする。Copilot バックエンド経由でも Claude モデルを選択することで意思決定粒度を揃える運用である。
- 開発フローは RFC 駆動を中心とし、軽量パス（`/fix` / `/qfix`）は RFC を省略しつつも PR マージまで自動化する変種として位置付ける。
- サービス仕様書は複数 RFC に分解され得る上位概念であり、`/adev` は仕様書 §2 の E2E テスト要件を最終ゴールとして全 RFC を横断的にドライブする。
