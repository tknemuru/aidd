# architecture.md — aidd システム構造オリエンテーション

## 本ドキュメントの位置付け

本ドキュメントは、aidd リポジトリで AI エージェントおよび新規参画者が効率的にオリエンテーションを行えるよう、ディレクトリ責務・コンポーネント責務分担・技術選定の背景を記述する。コードの挙動や関数レベルの実装詳細は対象外であり、それらはコードを直接参照して把握する。

## 提供価値

- リポジトリ横断で再利用可能な RFC 駆動開発フロー定義（スラッシュコマンド・テンプレート・ワークフロー）を一元管理する。
- 定義済み資産を配布 CLI で対象リポジトリに同期し、Claude Code / GitHub Copilot の双方を同一の運用で扱う。
- 自動開発オーケストレーションを単一コマンドで起動し、RFC 起草から実装マージまでを AI 主導で進める。

## 主要ディレクトリの責務

| ディレクトリ | 責務 |
| :--- | :--- |
| `bin/src/` | 配布 CLI とオーケストレータの TypeScript 実装の正本 |
| `bin/dist/` | TypeScript のビルド成果物（npm グローバルインストールで参照される） |
| `bin/test/` | 単体テストの TypeScript ソース |
| `adapters/claude/` | Claude Code 向け配布資産（指示書・ルール・テンプレート・ワークフロー） |
| `adapters/commands/` | 中立プロンプトマスタ。Claude / Copilot 双方へ配布されるスラッシュコマンド定義 |
| `docs/` | 本リポジトリ自身のシステム概要ドキュメント |
| `docs/rfcs/` | RFC ドキュメント |
| `registry/` | 将来的なマニフェスト管理領域（現状は空） |

## レイヤー構造

aidd の CLI 層は以下の 3 レイヤーから構成される。

| レイヤー | 責務 |
| :--- | :--- |
| エントリーポイント層 | `csync` / `rfc-init` / `rfc-publish` / `spec-init` / `adev` の 5 本 |
| オーケストレーション層 | フェーズ判定・リカバリループ・決定ログ等、AI 呼び出しの業務的組み立て |
| インフラ層 | AI バックエンド抽象、子プロセス起動、Git / gh 連携、ファイル同期 |

オーケストレーション層はインフラ層のみに依存し、エントリーポイント層はオーケストレーション層とインフラ層の双方を利用する。AI バックエンドの差し替えはインフラ層の `AiBackend` 実装追加のみで完結し、オーケストレーション層に変更は波及しない。

## コンポーネント責務分担

| コンポーネント | 責務 |
| :--- | :--- |
| AI バックエンド抽象 | ヘッドレス 1 問 1 答の契約を定義し、Claude / Copilot を切替可能にする |
| リカバリラッパ | 最大試行・タイムアウト・エスカレーションマーカー検出の共通ループ |
| 中立プロンプト変換 | 中立マスタを Claude 形式・Copilot 形式の双方に配布 |
| フェーズ検出器 | 成果物の実状態から現在のライフサイクルフェーズを判定 |
| 決定ログ | 自動開発中の意思決定を追跡可能な Markdown テーブルに追記 |
| 配布同期 (csync) | 対象リポジトリへの資産コピーとプロンプト変換を一括で実行 |

## 技術選定の背景

| 選定 | 理由 |
| :--- | :--- |
| Node.js / TypeScript | Windows ネイティブでの起動と型安全性を両立。Copilot CLI との親和性も高い |
| `node --test` | 外部依存を増やさず単体テストを実施。配布成果物への影響を局所化 |
| `simple-git` / `execa` を採用せず `spawn` 直接利用 | 依存最小化と Windows 側のシェル差異吸収を両立するため |
| シンボリックリンク不使用 | Windows 管理者権限要求を回避し、clone 体験を均質化するため |
| 中立プロンプトマスタの分離 | Claude / Copilot の双方向配布を 1 ソースから派生させるため |

## デプロイ構成

配布は npm のグローバルインストールを想定する。`package.json` の `bin` フィールドで 5 本の CLI 名を登録し、`npm install -g .` 実行時に Node.js のグローバル bin ディレクトリにシェル起動エントリが展開される。対象リポジトリ側の資産配置は `csync` コマンドで行い、aidd リポジトリへの実行時パス依存を残さない設計とする。
