## 1. 背景・動機 (Motivation)

<!-- 散文で記述。ただし5文以内。 -->

自動開発パイプラインから呼び出されるリカバリ付き AI 実行関数が、標準出力をパイプで接続している。AI CLI にはstdoutが非TTY（パイプ）の場合にプロセスが終了しない既知バグがあり、検証フェーズ等の長時間セッションでプロセス全体がハングする。パイプをファイルリダイレクトに置換することで、この既知バグの発動条件を排除する必要がある。

## 2. 機能要件

### 達成すべき要件

<!-- コード識別子禁止。機能名称またはドメイン用語で記述。 -->

| ID | 要件 |
|----|------|
| F-1 | リカバリ関数の標準入出力をパイプからリダイレクトに変更する |
| F-2 | タイムアウト安全弁（1時間）を設ける |
| F-3 | エスカレーション検出機構を維持する |
| F-4 | リトライ機構（最大3回）を維持する |
| F-5 | 呼び出し側インターフェースの互換性を維持する |

### やらないこと

- 呼び出し側スクリプトの変更
- AI CLI 本体のバグ修正
- タイムアウト値の動的変更機構

## 3. 非機能要件

### 達成すべき要件

| ID | 分類 | 要件 |
|----|------|------|
| NF-1 | 信頼性 | ハング発生率をゼロにする |
| NF-2 | 信頼性 | タイムアウト超過時に確実にプロセスを終了する |
| NF-3 | 互換性 | 既存の全呼び出し箇所（12箇所）で動作が変わらない |

### やらないこと

- パフォーマンス最適化（現行と同等で十分）
- ログ形式の変更

## 4. 実現方式

<!-- 機能要件・非機能要件をどう実現するかの高レベル方針。 -->

| 要件 | 実現方式 |
|------|----------|
| F-1 パイプ排除 | stdin をファイルリダイレクト、stdout をファイルリダイレクトに変更 |
| F-2 タイムアウト | コマンド実行にタイムアウトラッパーを付与（1時間） |
| F-3 エスカレーション検出 | 出力ファイルに対するマーカー文字列検索を維持 |
| F-4 リトライ維持 | ループ構造・最大試行回数を変更しない |
| F-5 互換性維持 | 関数末尾で出力ファイルの内容を標準出力に出力 |
| NF-1 ハング排除 | パイプ排除により非TTYバグの発動条件を根本除去 |
| NF-2 強制終了 | タイムアウト超過時にプロセスを強制killする |

## 5. 代替案の検討

<!-- §4の実現方式に対する代替案。最低2案を比較。 -->

| 案 | 概要 | 採否 | 決定的理由 |
|----|------|------|------------|
| A: ファイルリダイレクト | パイプを完全排除しリダイレクトに置換 | **採用** | 非TTYバグの発動条件を根本除去できる |
| B: プロセス置換 | パイプの代わりにプロセス置換を使用 | 却下 | プロセス置換もパイプを内部的に使用するため根本解決にならない |
| C: バックグラウンド実行+監視 | AI CLIをバックグラウンドで起動しPIDを監視 | 却下 | 実装が複雑化し、エラーハンドリングの信頼性が低下する |

## 6. 外部仕様 (External Specification)

<!-- ユーザ・運用者視点の振る舞い。箇条書き＋コード例。 -->

- 自動開発パイプラインの利用者から見た動作は変わらない
- リカバリ関数の戻り値・標準出力・標準エラーの仕様は維持される
  - 成功時: 終了コード 0、AI CLI の出力が標準出力に出る
  - 最大試行到達時: 終了コード 1
  - エスカレーション検出時: 終了コード 2
- タイムアウト超過時は新たに終了コード 1 で終了する
  - 標準エラーに「試行 N 失敗」のログが出力される

## 11. 詳細設計 (Detailed Design)

<!-- 箇条書き＋コード例。 -->
<!-- コードブロック間補足は2文以内の散文許可。 -->

- `bin/lib/git-utils.sh` の `run_claude_with_recovery` 関数を以下のように変更する

### 初回実行パスの変更

現行の初回実行パス:

```bash
cat "$tmpfile" | claude -p "$@" 2>"$errfile" | tee "$outfile"
```

変更後:

```bash
timeout 3600 claude -p "$@" < "$tmpfile" > "$outfile" 2>"$errfile"
```

- stdin: `cat "$tmpfile" |` を `< "$tmpfile"` に変更
- stdout: `| tee "$outfile"` を `> "$outfile"` に変更
- `timeout 3600` でハング時の最終安全弁を追加

### 修復実行パスの変更

現行の修復実行パス:

```bash
{ cat <<RECOVERY_EOF
...
RECOVERY_EOF
} | claude -p "$@" 2>"$errfile" | tee "$outfile"
```

変更後:

```bash
cat <<RECOVERY_EOF > "$tmpfile"
...
RECOVERY_EOF
timeout 3600 claude -p "$@" < "$tmpfile" > "$outfile" 2>"$errfile"
```

修復プロンプトを一時ファイルに書き込んでからリダイレクトで渡す。

### 出力の標準出力への転送

パイプ排除により `tee` が使えなくなるため、関数内の成功パスで `cat "$outfile"` を実行する。

```bash
if timeout 3600 claude -p "$@" < "$tmpfile" > "$outfile" 2>"$errfile"; then
  if grep -q "ESCALATION_REQUIRED" "$outfile"; then
    echo "[claude] エスカレーション検出" >&2
    cat "$outfile"
    return 2
  fi
  cat "$outfile"
  return 0
fi
```

- エスカレーション検出時も `cat "$outfile"` で出力を行う（現行と同一）
- 正常成功時に `cat "$outfile"` を追加して呼び出し元の `$()` キャプチャとの互換性を確保する

### デバッグログの維持

- `echo "[DEBUG] ..." >&2` のデバッグログは変更しない
- `echo "[claude] 試行 N/M" >&2` のログも変更しない

### 単体テスト仕様

| テスト対象 | 検証観点 |
|-----------|----------|
| 初回成功パス | リダイレクト方式で終了コード 0 を返すこと |
| 初回成功パス | 出力が標準出力に転送されること |
| 修復実行パス | エラー情報と元プロンプトが修復プロンプトに含まれること |
| 修復実行パス | リダイレクト方式で終了コード 0 を返すこと |
| エスカレーション検出 | マーカー検出時に終了コード 2 で即座に終了すること |
| エスカレーション検出 | リカバリループが中断されること |
| 最大試行到達 | 3回失敗後に終了コード 1 を返すこと |
| タイムアウト | 指定時間超過後にプロセスが強制終了されること |
