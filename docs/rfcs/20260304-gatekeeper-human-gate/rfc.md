## 1. 背景・動機 (Motivation)

<!-- 散文で記述。ただし5文以内。 -->

ゲートキーパーレビューの「固定出力テキスト」化を実運用で適用したが、AI 修正者が繰り返しバイパスしている。RFC テンプレートの HTML コメント内に禁止キーワードがガイドライン注釈として含まれており、キーワードスキャンが誤検出する。AI 修正者はキーワードの同義語置換だけで通過し、ゲートキーパー自身も表面的修正を見抜けず PASS を出す。根本原因は AI が「楽をしようとする力」に対して AI 同士のレビューだけでは抗いきれないことであり、人間の介在が不可欠である。

## 2. 機能要件

### 達成すべき要件

<!-- コード識別子禁止。機能名称またはドメイン用語で記述。 -->

| ID | 要件 |
|----|------|
| F-1 | 自動 RFC コマンドのゲートキーパー FAIL 時に |
|     | 絶対守ること違反または |
|     | 勝手な先送り違反の場合のみ |
|     | 人間承認ステップを導入する |
|     | （対応方針を1案提示し承認を待つ） |
| F-2 | 自動実装コマンドの実装ゲート FAIL 時に |
|     | 絶対守ること違反または |
|     | 勝手な先送り違反の場合のみ |
|     | 人間承認ステップを導入する |
| F-6 | ゲートキーパー FAIL 時出力に |
|     | NG 詳細引用（セクション番号・行番号・ |
|     | 該当文言の引用）を復活させる |
| F-7 | ゲートキーパー FAIL 時出力の対応行は |
|     | 固定対応文を維持する |
| F-8 | RFC レビューコマンド・実装レビューコマンドの |
|     | キーワード FAIL 上書き対象を |
|     | 対応行のみに限定する |
|     | （NG 引用はゲートキーパーのものを維持） |
| F-9 | RFC ゲート検証項目・実装ゲート検証項目の |
|     | 「FAIL 時の固定出力」を |
|     | 「FAIL 時の固定対応文」に名称変更する |
| F-10 | RFC テンプレートの HTML コメント内から |
|      | 禁止キーワードを除去し、 |
|      | ゲート検証項目への間接参照に置き換える |

### やらないこと

- ゲートの追加・削除
- キーワード→ゲート対応表の変更
- ゲートキーパーの判定ロジック自体の変更
- 対話型コマンド（手動 RFC レビュー、手動実装レビュー）
  への人間承認ステップ追加
  （対話型は既に人間が介在しているため不要）
- フォールバック排除・テスト品質・環境状態変更等の
  機械的に判定可能なゲートへの人間承認
  （これらは自動修正で十分）
- 検証 FAIL 時の人間承認
  （検証はゲートとは別の仕組みであり、
  自動修正で対応する）

## 6. 外部仕様 (External Specification)

<!-- ユーザ・運用者視点の振る舞い。箇条書き＋コード例。 -->

- 自動 RFC・自動実装コマンドでゲートキーパー FAIL 発生時、
  FAIL ゲートが絶対守ること違反または
  勝手な先送り違反の場合のみ
  AI が対応方針を1案提示して人間の承認を待つ
- 承認するとその方針で修正が実行される
- 上記以外のゲート FAIL および検証 FAIL は
  従来通り自動修正で進行する
- 人間承認時の報告フォーマット:

```
レビュー結果: FAIL（ラウンド N/8）

## FAIL 指摘
（NG 内容の要約）

## 対応方針
（決定した修正方針。1案のみ）

承認する場合はそのまま続行を指示してください。
```

- ゲートキーパーの FAIL 出力にセクション番号・行番号・
  該当文言の引用が復活する（対応行は固定対応文を維持）
- FAIL 出力形式（変更後）:

```
## GATE-X: （ゲート名）
**判定: FAIL**
- NG: （セクション番号・行番号・該当文言の引用。
  または不在事実の一文記述）
- 対応: （検証項目の「FAIL 時の固定対応文」を転記）
```

- RFC テンプレートの HTML コメントから
  禁止キーワードが除去され、
  ゲート検証項目への間接参照に置き換わる

## 11. 詳細設計 (Detailed Design)

<!-- 箇条書き＋コード例。 -->
<!-- コードブロック間補足は2文以内の散文許可。 -->

### 変更1: ゲートキーパー人格定義の FAIL テンプレート変更

`prompts/roles/gatekeeper.md` の FAIL 時出力テンプレートを以下に変更する。

変更前:

```
## GATE-X: （ゲート名）
**判定: FAIL**
- NG: （検証項目の「FAIL 時の固定出力」を転記）
```

変更後:

```
## GATE-X: （ゲート名）
**判定: FAIL**
- NG: （セクション番号・行番号・該当文言の引用。
  または不在事実の一文記述）
- 対応: （検証項目の「FAIL 時の固定対応文」を転記）
```

### 変更2: ゲート検証項目の名称変更

`prompts/criterias/gate-criteria-rfc.md` および `prompts/criterias/gate-criteria-impl.md` の各ゲートのセクション名「FAIL 時の固定出力」を「FAIL 時の固定対応文」に変更する。

変更後のセクション名:

