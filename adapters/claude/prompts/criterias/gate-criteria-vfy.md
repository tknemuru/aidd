# Verification ゲートキーパー検証項目

## GATE-V1: 結果値の健全性

### 基準 A（適合証明）
- verification-results.md の「結果」列が
  PASS または FAIL のみで構成されているか

### 基準 B（不適合排除）
- SKIP、N/A、その他の不正な値が
  1つでも存在しないか

### FAIL 時の固定対応文

不正な結果値を検出。
検証結果の「結果」列は PASS / FAIL の
2値のみが許可される。
SKIP・N/A・その他の値を使用した
テスト項目を再実行し、
PASS または FAIL で報告せよ。

## GATE-V2: 実行証跡の真正性

### 基準 A（適合証明）
- エビデンス列の記述が、仕様の実行手順で
  指定されたコマンドの実際の実行結果であるか
- RFC §7（または仕様書 §2）の実行手順と
  エビデンスの対応が取れているか

### 基準 B（不適合排除）
- ユニットテストファイル参照
  （.test.ts, .test.js, .spec.ts, .spec.js）が
  エビデンスに含まれていないか
- テストフレームワーク用語
  （vi.fn, vi.mock, jest.fn, jest.mock, vitest）が
  エビデンスに含まれていないか
- モック関連用語
  （モック, スタブ, スパイ）が
  エビデンスに含まれていないか
- 関数レベル直接呼び出し
  （直接実行:）が
  エビデンスに含まれていないか
- SKIP / N/A が
  エビデンスに含まれていないか

### FAIL 時の固定対応文

不正な実行証跡を検出。
エビデンスは仕様で指定された
実行手順の実際の実行結果でなければならない。
ユニットテスト結果の代用・
モックによる検証・
関数の直接呼び出しは許可しない。
仕様の実行手順に従い E2E テストを再実行し、
実際のコマンド出力をエビデンスとして記録せよ。
